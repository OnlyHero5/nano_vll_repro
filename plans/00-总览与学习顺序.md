# Day7 全量完成教案总览

## 1. 这套教案要解决什么问题

你当前的仓库已经有了 `Day3 ~ Day5` 的大量主干代码，但还没有稳定收敛成一个真正“可交付”的系统。最明显的问题有三类：

1. 当前 `HEAD` 本身就有断裂点，例如 `layers/linear.py` 语法不闭合。
2. README、`todo_list.md`、测试覆盖、源码实现之间状态不一致。
3. Day6/Day7 的高级特性和验收闭环没有真正落地。

所以这套教案不是简单讲解“vLLM 是什么”，而是带你把**当前仓库**一步一步补成 `todo_list.md` 里定义的 Day7 全量版本。

## 2. 这套教案依赖哪些参考实现

这套教案的代码不是凭空编的，写每一篇时都要同时对照三条基线：

- 你当前仓库里的已有实现
- `GeeeekExplorer/nano-vllm`
- Hugging Face `transformers` 中的 `Qwen3` 实现

我建议你在手敲时把下面三类页面同时打开：

- `https://github.com/GeeeekExplorer/nano-vllm`
- `https://github.com/GeeeekExplorer/nano-vllm/blob/main/nanovllm/engine/model_runner.py`
- `https://github.com/GeeeekExplorer/nano-vllm/blob/main/nanovllm/layers/linear.py`
- `https://github.com/huggingface/transformers/blob/main/src/transformers/models/qwen3/modeling_qwen3.py`

## 3. 学习顺序为什么这样设计

你不能一上来就写 Tensor Parallelism 或 CUDA Graph。原因很简单：当前仓库主干本身还没有收口，尤其是 `layers/linear.py`、Qwen3 权重映射、Sampler 和测试链路都还没闭环。

所以顺序必须是：

1. 先修主干断点
2. 再补齐单卡完整推理链路
3. 再进入 TP
4. 再进入 CUDA Graph
5. 最后做 benchmark、测试和 Day7 收口
